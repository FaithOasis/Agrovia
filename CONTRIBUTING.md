# Contributing to Agrovia

Thanks for your interest in contributing! Please follow these steps:

1. Fork the repository and create a feature branch from `main`.
2. Keep changes focused and small; one logical change per PR.
3. Run `npm install` then `npm run dev` to test locally.
4. Add tests if you add functionality (there are no tests yet).
5. Open a pull request describing the change and referencing any related issues.

Code style:
- Use existing project conventions (ESLint is configured).
- Run `npm run lint` before opening a PR.

By opening a PR you agree to follow the project's license and code of conduct.
